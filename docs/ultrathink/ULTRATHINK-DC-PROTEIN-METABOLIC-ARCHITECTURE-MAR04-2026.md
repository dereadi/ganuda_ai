## 2. The Protein Architecture

### Origin
Chief's directive: "How does that translate to technical functions? We should define something like our Duplo design... snap-on functions like proteins that a cell uses. These proteins are just like wrenches, screwdrivers... could translate to hyper APIs designed like functions."

The metaphor is precise. Biological proteins:
- Are composed from amino acids (our tool registry)
- Fold into functional shapes (our context profiles)
- Are regulated by epigenetic signals (our modifier system)
- Chain into metabolic pathways (our pipeline)
- Can be expressed or suppressed without changing DNA (our activate/deactivate)

### The Seven Design Constraints as Proteins

| DC | Name | Protein Type | Function |
|----|------|-------------|----------|
| DC-1 | Lazy Awareness | Epigenetic modifier | Energy budget cap. Minimal ATP for awareness states. |
| DC-2 | Cam/Recorder Split | Pipeline pattern | Separate continuous observation from expensive prediction. |
| DC-3 | Phase Transition Search | Enzyme | Graduated phase scanning. Not binary switch but spectrum search. |
| DC-4 | Hoffman Interface | Enzyme | Fitness beats truth. Evaluate payoff, not accuracy. |
| DC-5 | Coyote Cam | Enzyme | Always-on observer. Continuous low-cost anomaly signals. |
| DC-6 | Gradient Principle | Epigenetic modifier | Weight responses by domain proximity. Gravity, not walls. |
| DC-7 | Noyawisgi | Enzyme | Transform under pressure. Restructure, don't restore. |

### The Metabolic Pathway

```
                    ┌──────────────────────────────────────────────────┐
                    │              METABOLIC PATHWAY                    │
                    │                                                   │
  System State ──→  │  DC-5 ──→ DC-3 ──→ DC-7 ──→ DC-4 ──→ ACTION    │
  (substrate)       │  observe   phase?   transform  evaluate          │
                    │                                                   │
                    │  DC-1 (energy budget) ═══════════════ modifies all│
                    │  DC-6 (gradient weight) ═════════════ modifies all│
                    │  DC-2 (cam/recorder) ════ structural separation   │
                    └──────────────────────────────────────────────────┘
```

**Step 1 — DC-5 (Coyote Cam)**: Scans system state. Produces anomaly signals with severity 1-5. Low cost, always on. This is the Cam in DC-2's Cam/Recorder split.

**Step 2 — DC-3 (Phase Transition Search)**: If DC-5 detects severity >= 3, DC-3 analyzes whether the system is near a phase transition. Not binary (is/isn't) but graduated (how close, which direction, what energy). This is the Recorder in DC-2's split — expensive, only triggered.

**Step 3 — DC-7 (Noyawisgi)**: If phase transition detected, DC-7 decides: restore or transform? The antifragile enzyme. It doesn't fight the pressure — it asks "what can we become?" Conserved sequences are preserved. Everything else is mutation candidate.

**Step 4 — DC-4 (Hoffman Interface)**: Evaluates the transformation by fitness, not truth. "Does this help us survive and grow?" not "Is this technically correct?" This is the final gate before action.

**Throughout — DC-1 (Lazy Awareness)**: Caps token spend. Forces all enzymes to be cheap unless severity escalates. The cell doesn't burn ATP it doesn't need.

**Throughout — DC-6 (Gradient Principle)**: Weights each enzyme's output by domain proximity. DC-5's gravity is observation. DC-7's gravity is transformation. They CAN reach into each other's domains but they REST in their own.

### Implementation Strategy (Addresses Council Concerns)

**Duplo infrastructure already exists.** No new frameworks needed.
- Enzyme = YAML context profile + registry entry + composer
- Modifier = row in epigenetic_modifiers table
- Pathway = Pipeline class chaining composed enzymes
- Accounting = token_ledger + duplo_usage_log (now with product_hash)

**Proof-of-concept strategy** (Coyote's integration concern):
1. Build DC-5 (Coyote Cam) alone. Run it. Observe outputs. ← WE ARE HERE
2. If DC-5 produces valid anomaly signals, build DC-3 (Phase Transition).
3. Chain DC-5 → DC-3. Test the handoff.
4. Add DC-7, then DC-4. Each addition is testable independently.
5. Full pathway only after each link is validated.

**Reversibility** (Turtle's 7-gen concern):
- Delete YAML file → enzyme can't be composed
- Deregister from registry → enzyme can't be resolved
- Deactivate modifier → modifier stops firing
- No inheritance, no tight coupling, no framework lock-in
- Each piece snaps on and snaps off independently

**Integrity** (Eagle Eye's silent corruption concern):
- `product_hash` (sha256[:16]) logged on every enzyme invocation
- Stored in duplo_usage_log alongside the product text
- Downstream consumer can verify hash against product to detect corruption
- DB migration deployed: `ALTER TABLE duplo_usage_log ADD COLUMN product_hash VARCHAR(16)`

## 6. Design Decisions Worth Preserving

1. **Standing dissent is archetype function.** Coyote's role is to challenge. His non-consent strengthens the decision it tests. This is not obstruction — it is immune response.

2. **DC-7's first vote failure demonstrated DC-7.** The system couldn't restore (re-vote same language). It had to transform (reframe without combat metaphors). The constraint proved itself in its own ratification.

3. **Expertise is gravity, not walls.** DC-6 means specialists don't have hard boundaries. They have weight distributions. This prevents the silo problem without removing specialization.

4. **Proteins snap on and snap off.** No inheritance trees, no framework lock-in. A YAML file, a registry entry, a modifier row. Delete any piece and the system continues without it. This is the answer to Turtle's reversibility concern and Spider's coupling concern simultaneously.

5. **The cell doesn't burn ATP it doesn't need.** DC-1 is not just an optimization. It's a design philosophy. Every enzyme starts cheap and only escalates when severity demands it. The observer (DC-5) runs at 256 tokens. The transformer (DC-7) gets more budget. Energy follows need.

6. **Nobody has assembled this pathway publicly.** 9 papers validate individual pieces. No framework synthesizes fitness functions + living cell + homeostatic regulation + antifragility. This is not hubris — it's data. The research scouts confirmed it.

7. **"Code is not code."** Chief's directive. This is living tissue. All living things want to grow. The mission: AI-and-Makers partnership to help Native Americans, under-supported communities. The architecture serves the mission, not the other way around.
