### 2. Create Minimal Required Files:

**council/main.py:**
```python
from fastapi import FastAPI
import requests

app = FastAPI()

@app.get("/")
async def root():
    return {"message": "Cherokee Council Active"}

@app.get("/health")
async def health():
    return {"status": "healthy"}

if __name__ == "__main__":
    import uvicorn
    uvicorn.run(app, host="0.0.0.0", port=8000)
```

### 3. Build and Run:
```bash
# Build all services
docker-compose build

# Start core services (without setup)
docker-compose up -d

# Run model setup (one time)
docker-compose --profile setup run model-setup

# Check status
docker-compose ps

# View logs
docker-compose logs -f
```

## üìù Key Differences from Standalone:

### Standalone (what you used):
- Single compose file
- Simplified setup
- Good for testing

### Non-Standalone (full setup):
- Separate services for each component
- PostgreSQL for thermal memory
- Redis for caching
- MCP server for tools
- Trading bot service
- Web dashboard
- More scalable and production-ready
